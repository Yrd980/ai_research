# 如何把这批 AI 早报数据变成“可复利资产”

你现在有的是高价值“水上信号流”。  
目标是把它变成三层资产：**事实库 → 关系网 → 预测引擎**。

## 1. 数据分层（先稳结构）

### L0 原文层（不改写）
- 路径：`data/raw/wechat/`
- 作用：保真、可回溯、可审计
- 规则：每天一文件 + `ingest_manifest.csv` 状态跟踪

### L1 结构化事实层（事件表）
- 路径：`data/processed/juya_network_seed.csv`
- 粒度：每条新闻一行（date/item_no/entity/event/source/confidence）
- 规则：不做观点，只记录事实

### L2 语义层（词典 + 图谱）
- 词典：`data/processed/term_lexicon_0211_0209.csv`
- 图谱：`docs/framework/analysis_network.md` + 后续可转 `graph.csv`
- 规则：同名归并、别名归并、版本号保留

### L3 洞察层（可决策）
- 洞察：`docs/analysis/underwater_insights_0211_0209.md`
- 观察池：`data/processed/startup_watchlist_v1.csv`
- 规则：每条判断都要能回溯到 L1/L0 证据

## 2. 你可以立刻跑的 4 个高价值用法

### 用法 A：传闻兑现率追踪（最适合你）
- 输入：所有 `event_type in (Rumor, PreviewRelease)`
- 输出：`兑现天数、兑现率、落地版本偏差`
- 价值：把“全网扒信息”转成“谁最可靠”的量化资产

### 用法 B：公司动作强度评分
- 维度：发布频次 + API/生态动作 + 融资/并购动作
- 输出：周榜（公司热度、动作密度、资本强度）
- 价值：发现“低噪声高执行力团队”

### 用法 C：赛道温度雷达
- 赛道：Agent 编排、图像生成、语音生成、端侧量化、协议层(MCP/skills)
- 输出：7天/30天热度变化
- 价值：定位“正在升温但还没过热”的方向

### 用法 D：创始人-公司联动图
- 图谱关系：`FOUNDED_BY`、`RAISED`、`PARTNERED_WITH`、`ACQUIRED`
- 输出：人物中心性、组织扩张路径
- 价值：用“人”串起公司演化脉络

## 3. 把“模糊思路”落地成固定节奏（建议）

### 每日（10~20 分钟）
1. 原文入库（L0）
2. 事件抽取（L1）
3. 名词补全（L2）
4. 快速洞察 3 条（L3）

### 每周（1 小时）
1. 传闻兑现复盘
2. 初创观察池升降级（A/B/C）
3. 机会清单 Top 10 更新

### 每月（半天）
1. 做一次“预测 vs 现实”回测
2. 修正评分模型权重
3. 产出月度研究备忘录（可对外展示）
